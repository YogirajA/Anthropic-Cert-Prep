Domain 2, Tool Design and MCP Integration.

Section one, cheat sheet.

Domain 2: Tool Design & MCP Integration. Tool descriptions: The single biggest factor in tool selection performance. Aim for 3-4+ sentences covering what, when, when-NOT, params, edge cases. Overlapping minimal descriptions cause misrouting. Tool definition fields: required name (regex ^[a-zA-Z0-9_-]{1,64}$), description, input_schema. Optional: input_examples, cache_control, strict: true (guarantees schema conformance), defer_loading, allowed_callers. Structured errors: Return tool_result with is_error: true and a descriptive content string. Tool result blocks must immediately follow tool_use; results come first in user content (any text after). Claude retries 2-3x on parameter errors. tool_choice: {"type":"auto"} (default), {"type":"any"} (force any tool), {"type":"tool","name":"..."} (force specific), {"type":"none"}. Extended/adaptive thinking is incompatible with any and named tool: only auto or none. MCP transports: http (streamable-http, recommended for remote), sse (deprecated for remote in Claude Code), stdio (local processes). WebSocket is NOT in the 2025-11-25 MCP spec. MCP scopes: local (default, was "project") in about /.claude.json; project (shared via .mcp.json at repo root, version-controlled); user (was "global"). Precedence: local > project > user > plugin > connectors. Env var substitution: ${VAR} and ${VAR:-default} in command, args, env, url, headers. Plugins additionally expose ${CLAUDE_PLUGIN_ROOT} and ${CLAUDE_PLUGIN_DATA}. MCP tool naming: mcp__<server>__<tool> (double underscores). allowedTools accepts wildcards: mcp__github__*. Reserved name: workspace cannot be used as a server name. Tool granularity: 4-5 tools per agent target. Consolidate related ops into one tool with an action param rather than fragmenting into create_pr/review_pr/merge_pr; but avoid kitchen-sink tools too. Scope tools to role. Built-in API tools: text_editor_20250728 (commands: view, str_replace, create, insert; undo_edit removed; new max_characters param). computer_20251124 (Opus 4.5+/Sonnet 4.6, adds zoom action). Server tools: web_search_20260209, web_fetch, code_execution, tool_search. Claude Code built-ins: Grep (ripgrep content search), Glob (filename patterns), Read (full file incl. PDF/images/notebooks), Edit (unique-text replacement), Write (create/overwrite), Bash, Agent (was Task), WebFetch, WebSearch, ToolSearch (loads schemas on demand). Resources vs tools vs prompts (MCP): Resources = addressable read-mostly data (URIs, schemas, catalogs). Tools = actions/computations. Prompts = user-triggered templated workflows. Sampling = server asks client to call LLM (requires user approval).

Section two, quick review.

Description quality. #1 selection factor. 3-4+ sentences: what, when, when-NOT, params, edge cases. Differentiate similar tools clearly.

Tool definition. Required: name (regex ^[a-zA-Z0-9_-]{1,64}$), description, input_schema. Optional: input_examples, strict, cache_control.

tool_choice values. auto (may return text), any (must call SOME tool), {type:'tool',name:...} (force specific), none (no tools).

tool_choice + thinking. any and named-tool are INCOMPATIBLE with extended/adaptive thinking. Only auto/none allowed when thinking on.

Structured errors. Return tool_result with is_error: true + descriptive content. Include errorCategory (transient/validation/business/permission) and isRetryable. Claude retries 2-3x on param errors.

Tool count. 4-5 per agent target. >18 degrades selection. Scope tools to role; provide cross-role only for high-frequency needs.

Granularity. Consolidate related ops with an action param (avoid kitchen-sink AND micro-fragmentation). Split monoliths when truly distinct.

MCP transports. http (recommended for remote), sse (DEPRECATED for remote), stdio (local). WebSocket NOT in 2025-11-25 spec.

MCP scopes (RENAMED). local (default, was 'project') in about /.claude.json. project (shared via .mcp.json). user (was 'global').

Env var substitution. ${VAR} or ${VAR:-default} in command, args, env, url, headers. Plugin scope: ${CLAUDE_PLUGIN_ROOT}, ${CLAUDE_PLUGIN_DATA}.

MCP tool naming. mcp__<server>__<tool> (double underscores). Wildcards work in allowedTools (e.g. mcp__github__*). workspace reserved.

MCP roles. Resources = addressable read-mostly data (URIs, schemas, catalogs). Tools = actions/computations. Prompts = user-triggered templated workflows. Sampling = server asks client to call LLM.

Built-in CC tools. Grep (content), Glob (filenames), Read (full file incl. PDF/img), Edit (unique-text), Write, Bash, Agent, WebFetch, WebSearch, ToolSearch, AskUserQuestion.

AskUserQuestion. Built-in tool letting Claude pause in-loop to ask the user a multiple-choice question. Use for clarifying ambiguous intent or confirming destructive actions before proceeding.

strict tools. strict: true guarantees schema-valid input. Combine with tool_choice: 'any' for forced + valid. Limits: 20 strict tools, 24 optional params, 16 unions.

Community vs custom MCP. Prefer registered/community MCP for standard integrations (Jira, GitHub). Custom only for team-specific workflows. Vet third-party (prompt-injection vector).

Local error recovery. Subagents try transient retries locally. Only propagate errors they can't resolve (with what was attempted + partial results).

Minimal descriptions. "Retrieves customer info" vs "Retrieves order details" causes misrouting. Expand to 3-4+ sentences.

Generic 'failed' errors. "Operation failed" hides type, retryability. Always return structured error metadata.

Same shape access fail vs empty. Distinguish access failures (timeouts, perm denied) from valid empty results. Same shape hides info from coordinator.
