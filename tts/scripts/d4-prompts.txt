Domain 4, Prompt Engineering and Structured Output.

Section one, cheat sheet.

Domain 4: Prompt Engineering & Structured Output. Be specific, not vague: "Flag when claimed behavior contradicts code" beats "be conservative." Claude 4.x is more literal: vague hedges like "be thorough" or "high-confidence only" cause recall drops because the model now obeys. Multishot examples: 3-5 diverse examples (was 2-4), wrapped in <example> tags inside <examples>. Show ambiguous-case reasoning. Most effective when judgment matters most. XML tags: Anthropic-canonical structure: <instructions>, <context>, <input>, <documents>, <document index="n">, <document_content>, <source>, <thinking>, <answer>, <quotes>. Long context: Place documents at the TOP, query at the END (about 30% quality lift on multi-doc). Ground the model in <quotes> first. Adaptive thinking (replaces budget_tokens): thinking: {type: "adaptive"} + output_config.effort: low or medium or high or xhigh or max. Manual extended thinking (budget_tokens) is NOT supported on Opus 4.7, Opus 4.8, or Fable 5 / Mythos 5 (returns 400). Interleaved thinking is automatic on Opus 4.7/4.8/4.6. On Fable 5 / Mythos 5 thinking is always on (omit the param; even {type:"disabled"} 400s), whereas on Opus 4.7/4.8 adaptive is opt-in (off unless you set {type:"adaptive"}). Prefilling deprecated: Last-assistant-turn prefill is no longer supported on Claude 4.6 / 4.7 / 4.8 / Fable 5 / Mythos 5 (returns 400). Migrate to Structured Outputs, explicit "respond directly" instructions, XML wrappers, or tool calling. Structured Outputs (GA 2026): Native JSON-schema enforcement via output_config.format. SDK helpers: client.messages.parse() with Pydantic; zodOutputFormat(); outputConfig(Class<T>). Limits: 20 strict tools max, 24 optional params, 16 union-typed params. Incompatible with Citations (400 error). Strict tool use: strict: true on a tool guarantees schema-valid input. Combine with tool_choice: "any" to guarantee both a call and a valid schema. Make optional fields nullable to prevent hallucination. Schema redundancy: Include calculated_total + stated_total; flag mismatch for human review. Catches OCR/extraction errors automatically. Resilient enums: Add "other" + detail string field. Avoid fragile expansion as new types arrive. For amended docs, capture multiple values with source and effective_date. Batch API: 50% discount, 100,000 requests OR 256 MB per batch (whichever first), 24-hour expiry, results retained 29 days. custom_id for correlation. NOT ZDR-eligible. Supports vision, tool use, multi-turn, thinking, structured outputs. Use 1-hour cache (5-min often expires mid-batch). Citations: citations: {enabled: true} on each document block (all-or-none). Returns char_location, page_location, or content_block_location. cited_text is free. Works with prompt caching and Batch API. ZDR-eligible.

Section two, quick review.

Be specific not vague. Claude 4.x is LITERAL. "Be conservative" / "high-confidence only" / "be thorough" cause recall drops. Use concrete categorical criteria.

Few-shot count. 3-5 diverse examples in <example> tags inside <examples>. Show ambiguous-case reasoning.

XML tags. <instructions>, <context>, <documents>, <document index='n'>, <quotes>, <thinking>, <answer>.

Chain of Thought (CoT). Layer-1 reasoning primitive. Have the model explicitly reason step-by-step before answering. In Claude: <thinking>/<answer> tags for manual CoT, or extended/adaptive thinking (server-side, billed separately). Foundation for ReAct (CoT + tool call) and Reflexion (CoT applied to past actions).

Self-Consistency. Generate multiple completions for the same prompt, pick the majority answer. Reduces variance on hard reasoning. In Claude: run N parallel API calls (different temperature/seeds), aggregate results. Related to multi-pass review (independent reviewer + cross-file integration). Not heavily tested on CCA-F by name.

Long context placement. Documents at TOP, query at END. about 30% quality lift on multi-doc tasks. Ground in <quotes> first.

Adaptive thinking. thinking: {type: 'adaptive'} + output_config.effort. Replaces budget_tokens on 4.6/4.7/4.8 and Fable 5 / Mythos 5. Manual ext thinking returns 400 on Opus 4.7/4.8 and Fable 5.

Interleaved Thinking. Layer-2 pattern: thinking block runs BETWEEN every tool call (model reasons, calls tool, reasons again, calls next tool, etc). Automatic on Opus 4.8, Opus 4.7, Opus 4.6, and Sonnet 4.6 (and Fable 5 / Mythos 5) with adaptive thinking (no beta header). The interleaved-thinking-2025-05-14 beta header is only for older Claude 4 models (Opus 4.5/4.1/4, Sonnet 4.5/4). Lets budget_tokens exceed max_tokens.

Prefilling deprecated. Last-assistant-turn prefill no longer supported on Claude 4.6 / 4.7 / 4.8 and Fable 5 / Mythos 5 (returns 400). Migrate to Structured Outputs / explicit instructions / XML / tools.

Structured Outputs (GA). output_config.format. SDK helpers: messages.parse() (Python/Pydantic), zodOutputFormat() (TS), outputConfig(Class<T>) (Java). Limits: 20 strict tools max. Incompatible with Citations.

tool_use for JSON. Eliminates SYNTAX errors. Doesn't prevent SEMANTIC errors (totals not summing). Make optional fields nullable to prevent fabrication.

Schema redundancy. Include calculated_total + stated_total. Flag for human review only on mismatch. Catches OCR/extraction errors automatically.

Resilient enum. Add 'other' + detail string field. Avoid fragile enum expansion as new types arrive.

Validation retry. Works for FORMAT errors (resolved 2-3 attempts). Fails for MISSING information (data not in source). Recognize when to fail fast.

Batch API. 50% off, 100k requests OR 256 MB per batch (whichever first), 24h expiry, 29-day retention. custom_id for correlation. NOT ZDR-eligible.

Batch use cases. Overnight reports, weekly audits (yes). Blocking pre-merge checks (no: use sync). Match each API to its use case.

Multi-pass review. Per-file local pass + cross-file integration pass. Avoids attention dilution and contradictory findings on large PRs.

Citations. citations: {enabled: true} per document block (all-or-none). Returns char/page/content_block locations. cited_text free. Works with caching + Batch.

detected_pattern field. Track which patterns triggered findings then analyze false positives when developers dismiss findings.

Self-confidence filter. "Only report high-confidence findings" fails: LLM self-confidence poorly calibrated. Use specific categorical criteria.

Required when source lacks data. Forces fabrication. Make fields nullable so model can return null honestly.

Prefill on 4.6+. Returns 400. Don't prefill the last assistant turn on Claude 4.6 / 4.7 / 4.8 or Fable 5 / Mythos 5.
