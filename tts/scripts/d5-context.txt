Section two, quick review.

1M context (GA). Opus 4.8, Opus 4.7, Opus 4.6, Sonnet 4.6, and Fable 5 / Mythos 5: 1M tokens, no beta header, standard pricing. Haiku 4.5 = 200k. Sonnet 4.5 1M beta retired in 2026.

Lost in middle. Models reliably process start/end of long inputs but may miss middle. Place key summaries first, use section headers, ground in <quotes>.

Tool result accumulation. Trim 40+ fields then about 5 relevant. Persistent "case facts" block included each prompt. Extract structured facts before they enter history.

Escalation triggers. 3 valid: (1) explicit human request: honor immediately, (2) policy gap/silent, (3) inability to make progress. NOT sentiment, NOT self-confidence.

Error propagation. Return failure type, attempted action/query, partial results, alternative approaches. Distinguish access failure from valid empty result.

Conflicting sources. Annotate with attribution + publication dates. Don't pick one or average ($4.2B 2024 vs $3.8B 2023 likely temporal not contradictory).

Scratchpad files. Persist findings across context boundaries. Reference for subsequent questions. Counteracts context degradation in 30+ min sessions.

Memory tool (GA). memory_20250818, no beta header. CLIENT-side: you implement /memories backend. 6 commands: view, create, str_replace, insert, delete, rename. Validate paths against ../.

Agent Memory. Layer-2 pattern: durable state across sessions. In Claude: memory tool (memory_20250818) for cross-session facts; auto memory (Claude Code v2.1.59+) at about /.claude/projects/<repo>/memory/MEMORY.md; subagent memory frontmatter memory: project stored at ./.claude/agent-memory/<agent-name>/ (the user/local variants are referenced in some docs but path specifics may vary); scratchpad files for in-session persistence.

Compaction (beta). Header compact-2026-01-12. context_management.edits = [{type: 'compact_20260112'}]. Opus 4.8/4.7/4.6, Sonnet 4.6, Fable 5 / Mythos 5. NOT Haiku 4.5.

Prompt caching. 4 breakpoints max per request. Hierarchy: tools then system then messages (changes invalidate that level + all below).

Cache placement. Place cache_control on the LAST stable block (not per-request user content). 20-block lookback for cache reads.

Cache TTL. 5-min default ({type:'ephemeral'}) or 1-hour ({ttl:'1h'}). 1h breakpoints MUST appear before 5m or API errors. Reads 0.1x, 5m write 1.25x, 1h write 2x.

Cache minimums. Fable 5 / Mythos 5 = 512. Opus 4.8 = 1024. Opus 4.7 = 2048. Opus 4.6 / 4.5 = 4096. Sonnet 4.6 / 4.5 = 1024. Haiku 4.5 = 4096. Below-min caches silently: verify via usage fields.

pause_turn. Server tools (web_search/web_fetch/code_execution) hit 10-iter sampling cap. Re-send the assistant response unchanged to continue. Not an error.

Stale tool_results on resume. Filter previous tool_result messages on resume; keep human/assistant turns. Forces fresh data fetch.

Confidence calibration. Validate accuracy by document type AND field. 97% aggregate can mask 80% on contracts. Stratified random sampling.

Provenance. Subagents output structured claim/evidence/source/confidence/dates. Synthesis preserves attribution: don't compress it away.

Context engineering. Anthropic's 3 pillars: just-in-time retrieval, structured note-taking (memory/files), sub-agent isolation. Treat context as precious.

Silent error suppression. Returning empty results as success hides failures from coordinator. Never suppress: return structured failure context.

Cache_control on user msg. On per-request user content: cache key changes every turn then 0% hit rate. Place on stable prefix.

Sentiment-based escalation. Sentiment doesn't correlate with case complexity. Use explicit categorical criteria.

Self-confidence escalation. LLM self-confidence is poorly calibrated: agent already incorrectly confident on hard cases. Use the 3 explicit triggers.
