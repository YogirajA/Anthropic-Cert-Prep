Section one, cheat sheet.

Exam Reference: Official Scope & Structure. Source: Anthropic CCA-F Exam Guide v0.1 (Feb 10 2025) + 2026 doc refresh. Scaled score 100-1000, pass at 720. Multiple choice, 1 correct + 3 distractors. Unanswered = incorrect (no guessing penalty). Domain weights: D1 Agentic Architecture (27%) , D2 Tool Design and MCP (18%) , D3 Claude Code Config (20%) , D4 Prompt Engineering (20%) , D5 Context and Reliability (15%). Six official scenarios (4 picked at random per exam):. 1. Customer Support Resolution Agent: MCP tools (get_customer, lookup_order, process_refund, escalate_to_human); 80%+ first-contact resolution target. 2. Code Generation with Claude Code: CLAUDE.md, custom slash commands, plan mode vs direct execution. 3. Multi-Agent Research System: Coordinator + 4 subagents (web search, document analysis, synthesis, report); cited reports. 4. Developer Productivity: Built-in tools (Read, Write, Bash, Grep, Glob) + MCP servers; codebase exploration. 5. Claude Code for CI/CD: Automated reviews, test generation, PR feedback; minimize false positives. 6. Structured Data Extraction: JSON schemas, validation, edge cases, downstream integration. In-scope (study): Agentic loop and stop_reason, multi-agent orchestration, subagent context passing, tool descriptions, MCP scopes and env vars, CLAUDE.md hierarchy, .claude/rules/ glob patterns, skills (context: fork, allowed-tools, argument-hint), plan mode, -p / --output-format json / --json-schema, tool_use schemas, tool_choice, few-shot, Batch API, lost-in-the-middle, escalation triggers, structured error propagation, scratchpad files, source provenance, confidence calibration. Out-of-scope (do NOT study): Fine-tuning and training, API auth/billing, MCP server hosting/infra, model internals/RLHF, embeddings/vector DB internals, computer use, vision/image analysis, streaming API, rate limits/pricing, OAuth/API-key rotation details, cloud provider configs, prompt-caching implementation details (beyond "knowing it exists"), token counting. Note on terminology: The exam baseline (Feb 2025) calls subagent spawning the Task tool. As of Claude Code v2.1.63 it's the Agent tool; both names coexist. Either is acceptable. Same for "Claude Code SDK" equals "Claude Agent SDK." The 2026 reference card below covers post-baseline doc evolution that may surface in updated exam forms. ⚠ The exam is model-version-AGNOSTIC. It tests architectural judgment, the agentic loop, MCP/SDK mechanics, prompting and context patterns — not which model is newest. As of June 2026 the lineup is Fable 5 and Mythos 5 (most capable; Mythos via Project Glasswing) and Opus 4.8 (current default), above Opus 4.7/4.6, Sonnet 4.6, Haiku 4.5 — but you will not be asked to recall Fable-vs-Opus benchmarks or release dates. Know the lineup for context; spend study time on the patterns. The model facts below (adaptive thinking, prefill removal, compaction, xhigh) are framed on the 4.6/4.7 family and apply equally to Opus 4.8 and Fable 5 / Mythos 5.
