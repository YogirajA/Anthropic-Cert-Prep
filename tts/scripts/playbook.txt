Architect's Playbook: Design Patterns & Anti-Patterns. 4 Constraint Dimensions (third-party framing): Latency then parallelization and caching. Accuracy then structured intermediates and few-shot. Cost then Batch APIs and context pruning. Compliance then application-layer intercepts (NEVER prompts). Anthropic explicitly names only the latency/cost tradeoff in "Building Effective Agents"; accuracy and compliance are scaffolding added by the third-party Playbook. Zero-Tolerance Compliance: Emphatic prompts have a non-zero failure rate. Use hooks (PreToolUse then permissionDecision: "deny") or tool_choice to enforce. Block server-side, invoke escalation. Remove model discretion. Tool Context Pruning: Filter verbose tool responses (40+ fields then about 5 relevant) application-side before they enter context. Essential for multi-issue support sessions. Session Resumption: Filter out stale tool_result messages on resume; keep human/assistant turns. Forces agent to re-fetch current data. For dynamic environments, inform agent which files changed. Scratchpad Pattern: Persist key findings, architectural maps, decisions in scratchpad.md. Reference for subsequent questions. Counteracts context degradation in 30+ minute sessions. Pairs with the Memory tool for cross-session persistence. Lead-Coordinator State Management (corrected from Anthropic's multi-agent research blog): Subagents do NOT share memory or read each other's state. They return findings to the lead/coordinator only. The LEAD persists its plan to memory/filesystem to survive long contexts; subagents write outputs to a filesystem and pass lightweight references back to the lead to minimize information loss during multi-stage processing (the blog's framing). Avoid daisy-chaining full conversation logs (which scales tokens exponentially): pass concise structured findings instead. Note: the third-party "shared vector store between subagents with semantic search" is NOT Anthropic-recommended; their architecture is lead-coordinator-only. Forced Execution Order: Use tool_choice: {"type":"tool","name":"extract_metadata"} on first API call. Don't rely on prompt begging: use API constraints. Note: incompatible with adaptive/extended thinking. Goal-Oriented Delegation: Specify goals + quality criteria, not step-by-step procedures. Let subagents determine strategy. Add enum parameters (analysis_type: extraction or summarization) to guide behavior. MCP Tool Granularity (right-size, don't over-split): Anthropic's "Writing Tools for Agents" recommends consolidating related ops (e.g., schedule_event combining list_users/list_events/create_event). Split into focused tools when descriptions overlap; consolidate when steps belong to one workflow. Improving descriptions often beats splitting. Agents default to familiar built-ins (Grep) when custom tools are vaguely described. Structured Intermediates: Standardize multi-format subagent outputs to common JSON (claim, evidence, source, confidence) via format conversion layer. Preserve claim-source mappings through synthesis. Schema Redundancy: calculated_total + stated_total. Flag mismatch for human review. Catches OCR and extraction errors automatically. Resilient Catch-Alls: Enum + "other" + detail string. Avoid fragile enum expansion. For amended docs, capture multiple values with source and effective_date. Memory + Compaction Pattern: For long-running agents: enable server-side compaction beta (summarizes prior turns) + memory tool (writes durable facts to /memories/). Compaction handles "what happened"; memory handles "what to remember next time.". 1M Context Caveats: 1M GA on Opus 4.8/4.7/4.6, Sonnet 4.6, and Fable 5 / Mythos 5 with no premium. Still subject to lost-in-the-middle and context rot. Use prompt caching + just-in-time retrieval. Don't dump everything; load on demand. 8 Agent Design Patterns (framework view): Common framework names mapped to Anthropic implementation. Useful as study scaffolding; the CCA-F exam uses Anthropic's own terminology, not these names (only "scratchpad" and "plan mode" appear in the official guide by name). Chain of Thought then <thinking>/<answer> XML tags or extended/adaptive thinking. Anthropic name: extended thinking / adaptive thinking. 3. Self-Consistency then sample N completions, take majority. Not in CCA-F by name; closest: independent reviewer + multi-pass review. Layer 2: Agent loop patterns (loop, tools, persistent state):. 4. ReAct then reason then call tool, repeat. Section two, quick review.

4 constraint dimensions. Latency then parallelize + cache. Accuracy then structured + few-shot. Cost then Batch + pruning. Compliance then app-layer (NEVER prompts).

Zero-tolerance compliance. Hooks intercept tool calls server-side. Emphatic prompts have a non-zero failure rate. Remove model discretion entirely.

Tool context pruning. Filter verbose responses (40+ fields then about 5 relevant) app-side BEFORE they enter context.

Session resumption. Filter stale tool_result, keep human/asst turns. For dynamic envs, inform agent which files changed.

Forced execution order. tool_choice: {type:'tool',name:...} on first call. Caveat: incompatible with thinking: only auto/none allowed there.

Goal-oriented delegation. Specify GOALS + quality criteria, not procedures. Add enum params (analysis_type) to guide behavior.

MCP granularity. Right-size to the agent's role. Per Anthropic's "Writing Tools for Agents," consolidate related ops by default (e.g., schedule_event over list_users/create_event). Split only when descriptions overlap. Improving descriptions often beats splitting.

Lead-coordinator state. Per Anthropic's multi-agent research blog: subagents do NOT share memory; they return findings to the lead. LEAD saves its plan to memory/filesystem to survive long context; subagents output to filesystem and pass lightweight references back. Avoid daisy-chaining full logs (exponential cost). Note: "shared vector store between subagents" is third-party framing, NOT Anthropic-canonical.

Memory + Compaction. Server compaction summarizes prior turns + memory tool persists durable facts. Together unblock 30+ min sessions.

Schema redundancy. calculated_total + stated_total. Flag mismatch for human review. Catches OCR / extraction errors.

Resilient catch-all. Enum + 'other' + detail string. Avoid fragile enum expansion.

Structured intermediates. Standardize multi-format subagent outputs to common JSON (claim/evidence/source/confidence) via format conversion layer.

Daisy-chained logs. Full conversation logs between subagents = exponential token cost. Use a filesystem/artifact store: subagents write outputs and pass lightweight references back to the lead.

Procedural micromanagement. Step-by-step instructions to subagent ("Search X, Read Y"): rigid, misses emerging topics. Specify goals.

Read-loading codebase. Sequential Read of thousands of lines is context-heavy. Pattern: Glob/Grep first then pinpoint then Read targeted.

End of section.
